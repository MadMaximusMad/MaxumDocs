# Welcome to Maxum

Maxum is a treasury-backed base asset and liquidity engine designed for on-chain economies.

The system converts transaction activity into balance sheet growth through a closed-loop capital structure. Every interaction—whether trading, liquidity provision, bonding, or staking—feeds into a continuous process that reinforces liquidity, expands the treasury, and increases the system’s capacity to support future activity.

At its core, Maxum operates as a closed system:

```
Capital → Liquidity → Activity → Fees → Treasury → Liquidity
```

Capital enters the system and is deployed into liquidity. Liquidity enables markets, and markets generate activity. Activity produces transaction fees, which are redistributed across supply reduction, liquidity providers, and the treasury. Treasury growth increases the system’s balance sheet, allowing capital to be redeployed into liquidity and operations.

This loop is not descriptive—it is enforced. Every transaction contributes to it. Every interaction reinforces it.

Growth is not subsidized. It is not externally maintained. It is derived from activity and retained within the system.

MAX functions as the base asset within this structure. It is not only traded, but used as the unit of liquidity, settlement, and expansion. Supply is not fixed; it is constrained by the treasury’s Risk-Free Value (RFV), ensuring that circulating supply remains aligned with the system’s balance sheet.

As activity increases, value capture increases. As value capture increases, the balance sheet expands. As the balance sheet expands, the system’s capacity to support liquidity and activity increases.

Maxum extends beyond trading environments. MAX is designed to function as a base asset across token launch systems, gaming economies, and social interactions. These environments are throughput layers. They generate continuous transaction flow, and that flow is captured and reintegrated into the system.
